# PublishEA - OCTO Architecture Resources

## Overview
This repository contains the PublishEA project, which is designed to facilitate architectural resources and frameworks for the OCTO team.

## Purpose
The purpose of the PublishEA project is to provide a cohesive set of resources that can be utilized by team members for effective architectural decision-making and documentation.

## Key Features
- Comprehensive resource library
- Easy access to architectural frameworks
- User-friendly documentation

## Project Structure
The project is organized into several key directories:
- `src/`: Contains the source code
- `docs/`: Contains documentation
- `tests/`: Contains unit tests

## File Descriptions
- `README.md`: This file provides an overview of the project.
- `main.py`: The main program file.
- `config.yaml`: Configuration file for the application.

## Current Status
The project is currently in active development. Features are being added based on team feedback and project requirements.

## Getting Started
To get started with the PublishEA project, please follow these steps:
1. Clone the repository: `git clone https://github.com/govuk-digital-backbone/publish-ea.git`
2. Install the required dependencies: `pip install -r requirements.txt`
3. Run the application: `python main.py`

## Development
For development purposes, please adhere to the following guidelines:
- Ensure code is properly documented.
- Write unit tests for new features.
- Follow the coding standards outlined in `CONTRIBUTING.md`.

## Deployment
To deploy the application, follow these instructions:
1. Build the application with: `docker build -t publish-ea .`
2. Run the Docker container: `docker run -d publish-ea`
